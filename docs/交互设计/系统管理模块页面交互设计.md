## 1. 总体布局

### 1.1 管理端整体框架

管理端采用“左右结构”布局：

- **左侧**：固定宽度（240px）深色侧边栏，包含 Logo 和垂直导航菜单。
- **右侧**：自适应宽度内容区，分为顶部导航栏和主工作区。

```
┌──────────────────────────────────────────────────┐
│  Logo         管理端标题          [管理员] [退出] │ ← 顶部栏
├──────────┬───────────────────────────────────────┤
│ 仪表盘   │                                       │
│ 用户管理 │         主工作区                       │
│ 工作空间 │      (根据路由切换内容)                │
│ 角色管理 │                                       │
│          │                                       │
├──────────┴───────────────────────────────────────┤
```

### 1.2 全局交互规范

- **表格操作**：危险操作（禁用、删除、解散）均需二次确认弹窗。
- **批量操作**：表格支持多选，选中后底部浮现操作栏。
- **消息反馈**：成功/失败操作使用 Toast 提示（顶部居中，3秒自动消失）。
- **表单校验**：前端实时校验并提示错误信息，提交时再次全量校验。
- **加载状态**：数据请求时表格显示骨架屏，按钮显示 loading 状态。

| 操作    | 触发方式            | 反馈                  |

## 5. 角色与权限管理

**路由**：`/admin/roles`

### 5.1 整体布局

页面采用左右分栏结构：

- **左侧**：角色类型树，宽度约260px，可调整。
- **右侧**：选中角色的详情面板，包含“权限配置”和“关联用户”两个Tab页。

```
┌──────────────────────────────────────────────────┐
│  角色管理                                        │
├─────────────┬────────────────────────────────────┤
│ 角色树      │  角色详情（tabs）                   │
│             │  [权限配置] [关联用户]              │
│ 📁 系统角色 │  ┌───────────────────────────┐     │
│  ├ 系统管理员│  │ 操作对象 │ 权限点(复选框) │     │
│  ├ 用户管理 │  │ 用户管理 │ ☑查看 ☑创建   │     │
│  └ 空间查看 │  │          │ ☐编辑 ☐禁用   │     │
│             │  │ 工作空间 │ ☑查看 ☐创建   │     │
│ [+添加]    │  │ ...     │ ...           │     │
│             │  └───────────────────────────┘     │
│             │  [撤销修改] [保存权限]             │
└─────────────┴────────────────────────────────────┘
```

### 5.2 左侧角色树交互

#### 5.2.1 树节点结构

- 根节点按角色类型分组（如“系统角色”），类型节点不可编辑、不可删除。
- 根节点下挂载具体角色节点，显示角色名称，支持内联操作。

#### 5.2.2 添加角色

- 点击根节点（如“系统角色”）右侧的 **[+]** 按钮，在对应类型下新增一个可编辑节点。
- 新增节点默认名称为空，输入框自动聚焦，用户输入角色名称。
- 输入框失去焦点后，前端校验名称是否为空及唯一性，校验通过则调用创建角色接口，提交 `name` 和 `type='system'`，`permissions` 初始为空数组。
- 创建成功后节点保留在树中，选中新节点可继续编辑权限。

#### 5.2.3 编辑角色名称

- 双击树节点，节点名称变为可编辑输入框。
- 修改文本后，输入框失去焦点，自动调用更新角色接口，提交新名称。
- 若名称校验失败（重复或为空），输入框重新聚焦并提示错误。

#### 5.2.4 删除角色

- 右键点击角色节点或选中后点击树工具栏的删除按钮，弹出二次确认。
- 系统预置角色（isSystem=true）不可删除，隐藏删除入口。
- 若角色被用户引用，弹出提示列出引用用户，阻止删除。
- 删除成功后，右侧清空详情，树节点移除。

#### 5.2.5 选择角色

- **单击**树节点，右侧加载该角色的详情面板，默认显示“权限配置”Tab。

### 5.3 右侧角色详情面板

#### 5.3.1 权限配置Tab

- 以**表格**形式展示权限点，表格包含两列：
  - **操作对象**：权限点所属的功能模块，例如“用户管理”、“工作空间管理”、“角色管理”等，同一模块的权限点合并为一个单元格。
  - **权限点**：该模块下所有的具体权限点，每个权限点名称后带有一个复选框。复选框勾选表示拥有该权限。
- 表格示例如下：

| 操作对象   | 权限点                                                |
| ------ | -------------------------------------------------- |
| 用户管理   | ☑ 查看用户   ☑ 创建用户   ☐ 编辑用户   ☐ 禁用/启用用户   ☐ 重置密码      |
| 工作空间管理 | ☑ 查看工作空间   ☐ 创建工作空间   ☐ 编辑工作空间   ☐ 解散工作空间   ☐ 管理成员 |
| 角色管理   | ☑ 查看角色   ☐ 创建角色   ☐ 编辑角色   ☐ 删除角色                  |

- **勾选规则**：
  - 每个权限点独立控制，点击复选框即可切换选中/取消状态。
  - 系统预置角色（如“系统管理员”）的已有权限对应的复选框为禁用状态（灰显，不可取消勾选），以保持核心权限不被误移除。新增的权限点可正常勾选。
- **撤销修改**：在未点击“保存权限”按钮前，可点击[撤销修改]按钮，将权限配置恢复到上次保存时的状态。本次所有未保存的修改会被丢弃。
- **保存权限**：点击[保存权限]按钮，收集所有选中的权限点code，调用更新角色接口。保存成功后Toast提示“权限已保存”，并更新权限快照。

#### 5.3.2 关联用户Tab

- 展示拥有当前角色的用户列表，分页显示（用户名、邮箱、状态、操作）。
- **添加用户**：点击[添加用户]按钮，弹出多选搜索弹窗，可搜索平台用户（仅显示活跃用户），选择后批量插入 `user_role` 关联。
- **移除用户**：每行用户有[移除]按钮，点击后二次确认，解除该用户与当前角色的关联。
- 操作完成后列表自动刷新。

## 6. 通用交互模式总结

### 6.1 弹窗类型

| 类型   | 使用场景               |
| ---- | ------------------ |
| 确认弹窗 | 删除、禁用、解散、移除关联等危险操作 |
| 表单弹窗 | 快速创建（如新建工作空间、添加成员） |
| 选择弹窗 | 多选用户关联角色、选择成员      |

### 6.2 加载状态

- **列表加载**：首次显示骨架屏，刷新显示顶部进度条。
- **表单提交**：按钮loading+禁用，完成恢复。
- **树节点操作**：添加/编辑/删除时节点局部loading或全局提示。

### 6.3 响应式处理

- 侧边栏小屏幕下可折叠为图标模式。
- 角色管理左右分栏在小屏幕下切换为上下布局，树组件可折叠。
