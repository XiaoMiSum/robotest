## 1. 总体布局

### 1.1 项目工作区整体框架

进入项目后采用“顶部导航 + 内容区”布局。内容区根据功能模块不同采用不同结构：

- **功能测试**：左右分栏（左侧菜单导航 + 右侧工作区）
- **接口测试**：暂未开放，显示占位提示
- **缺陷管理**：独立功能面板

```
┌──────────────────────────────────────────────────────────┐
│  顶部导航栏                                              │
│  [Logo] [功能测试] [接口测试] [缺陷管理]                  │
│  [📁 我的空间] [📂 我的项目] [⚙ 空间管理] [🔧 系统管理] │
├──────────────────────────────────────────────────────────┤
│                                                          │
│                      内容区                               │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 1.2 顶部导航栏

项目工作区顶部导航栏分为两行：

- **第一行**：动态菜单区，包含项目内功能入口
- **第二行**：右侧图标组，与全局导航保持一致

#### 1.2.1 动态菜单

项目模式下动态菜单固定为三个入口：

| 菜单项  | 路由                             | 说明           |
| ---- | ------------------------------ | ------------ |
| 功能测试 | `/workspace/projects/cases`    | 测试用例管理、评审、计划 |
| 接口测试 | `/workspace/projects/api-test` | 暂未开放         |
| 缺陷管理 | `/workspace/projects/bugs`     | 缺陷全生命周期管理    |

当前激活菜单项高亮显示。

#### 1.2.2 默认进入

用户进入项目后，默认跳转至**项目工作台**页面（`/workspace/projects/dashboard`），展示项目概览信息。

#### 1.2.3 我的项目

点击顶部导航栏右侧“📂 我的项目”图标，跳转至我的项目页面。该页面展示当前工作空间下所有活跃项目列表，用户可切换或选择项目。

### 1.3 全局交互规范

- **消息反馈**：成功/失败操作使用 Toast 提示（顶部居中，3秒自动消失）。
- **危险操作**：删除、归档等需二次确认弹窗。
- **加载状态**：数据请求时显示骨架屏，按钮显示 loading 状态。
- **自动保存**：脑图编辑操作实时同步，无需手动保存。
- **实时协作**：编辑脑图时自动建立 WebSocket 连接，断线时顶部显示“正在重连...”横幅。

## 3. 功能测试

功能测试工作区采用**左右分栏**布局：

- **左侧**：菜单导航，展示测试用例、测试评审、测试计划三个子模块入口。
- **右侧**：子模块内容区，根据当前选中菜单项切换内容。

```
┌──────────────────────────────────────────────────┐
│  功能测试                                        │
├──────────┬───────────────────────────────────────┤
│ 菜单导航 │         子模块内容区                    │
│          │                                       │
│ 测试用例 │      (根据菜单切换)                     │
│ 测试评审 │                                       │
│ 测试计划 │                                       │
│          │                                       │
└──────────┴───────────────────────────────────────┘
```

### 3.1 左侧菜单导航

菜单项列表：

| 菜单项  | 图标  | 说明                 |
| ---- | --- | ------------------ |
| 测试用例 | 📋  | 管理测试用例，包含模块树和脑图编辑器 |
| 测试评审 | ✅   | 管理测试评审，包含评审列表和评审详情 |
| 测试计划 | 📝  | 管理测试计划，包含计划列表和执行跟踪 |

- 当前激活菜单项高亮背景 + 左侧指示条。
- 点击菜单项切换右侧子模块内容。
- 进入功能测试时默认选中“测试用例”。

### 3.2 测试用例子模块

选中“测试用例”菜单后，右侧进一步分为左右结构：左侧模块树 + 右侧脑图编辑器。

#### 3.2.1 页面布局

```
┌──────────┬───────────────────────────────────────┐
│ 模块树   │   📄 登录用例集                        │
│          │                                       │
│ 📁功能   │  ┌──────────────────────────────┐     │
│  📁登录  │  │        脑图编辑区              │     │
│   📄用例集│  │                              │     │
│  📁支付  │  │    ┌───┐                      │     │
│   📄流程 │  │    │根节点│                    │     │
│ [+新建]  │  │    └─┬─┘                      │     │
│          │  │  ┌───┴───┐                    │     │
│          │  │  │子节点1 │                    │     │
│          │  │  └───────┘                    │     │
│          │  └──────────────────────────────┘     │
```

#### 3.2.2 交互说明

**模块树交互**：展开/折叠点击箭头；选中文档点击节点加载脑图；新建右键或[+新建]输入名称；重命名双击或右键；拖拽移动节点；删除右键或Delete键二次确认。

**脑图编辑器**：添加子节点Tab；添加兄弟节点Enter；编辑文本双击；移动拖拽；删除Delete；标记用例/优先级工具栏或右键菜单；缩放Ctrl+=/-/0或工具栏按钮；抓手模式工具栏切换或按H键。

**脑图组件详细设计**参见《脑图组件实现设计文档》。

| 操作     | 触发方式       | 反馈                                                   |
| ------ | ---------- | ---------------------------------------------------- |

| 编辑字段   | 直接修改表单控件   | 非关闭状态下所有字段可编辑                                        |
| 更新状态   | 下拉选择新状态    | 状态变更将记录到操作日志                                         |
| 保存     | 点击[保存]     | 调用 `PUT /api/project/bugs/:bugId` → 操作日志刷新 → Toast提示 |
| 取消     | 点击[取消]或←返回 | 不保存离开（有修改时二次确认）                                      |
| 查看操作记录 | 滚动下方列表     | 按时间倒序展示所有操作历史                                        |
| 关闭状态限制 | 状态为“已关闭”时  | 所有表单控件变为只读文本，不显示[保存][取消]按钮，操作记录正常显示                  |

**状态流转规则**：

- 缺陷状态按“新建 → 已指派 → 修复中 → 已修复 → 已验证 → 关闭”流转。
- 关闭状态为终态，关闭后不可再修改任何信息。
- 可从任意非关闭状态重新打开（状态变更为“新建”或“已指派”）。

## 8. 通用交互模式总结

### 8.1 弹窗类型

| 类型    | 使用场景                 |
| ----- | -------------------- |
| 确认弹窗  | 删除节点/文档/模块、关闭计划、完成评审 |
| 表单弹窗  | 发起评审、创建计划、提交缺陷       |
| 选择器弹窗 | 关联用例、选择处理人、选择计划      |

### 8.2 状态标识规范

- **用例优先级**：P0红色、P1橙色、P2蓝色、P3灰色
- **执行结果**：通过🟢、失败🔴、阻塞⚪、未执行🟡
- **缺陷等级**：致命🔴、严重🟠、一般🟡、轻微🟢
- **评审标记**：通过✅、不通过❌

### 8.3 空状态处理

| 场景     | 展示                    |
| ------ | --------------------- |
| 无模块    | “暂无模块，右键或点击[+新建]创建”   |
| 文档无节点  | 显示根节点，提示“Tab添加子节点”    |
| 无评审/计划 | “暂无评审/计划，点击按钮创建”      |
| 无缺陷    | “暂无缺陷，点击[提交缺陷]创建”     |
| 脑图加载   | 文档区域显示骨架屏             |
| 协作断线   | 顶部黄色横幅“连接已断开，正在重连...” |
