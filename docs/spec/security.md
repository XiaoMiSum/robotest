# 工程规范 — 安全

**文档版本**：V1.0
**日期**：2026-07-06
**状态**：已发布

---

## 1. 认证与授权

- 所有非公开接口通过 JWT 认证，Token 通过 `Authorization: Bearer` 头传递。
- 管理端接口要求用户拥有至少一个系统角色。
- 业务端接口强制校验 `X-Active-Workspace` 头，验证用户工作空间归属。
- 项目内操作另需 `X-Active-Project` 头，验证项目归属。
- 业务端请求经由 `WorkspaceRoleFilter` 处理：读取 `X-Active-Workspace` 头，查询 `workspace_user` + `sys_role`，将空间角色对应的权限码注入 `LoginUser.authorities`，供后续 `@PreAuthorize` 等注解使用。
- 角色或权限变更后，强制 Token 失效（通过 Redis 黑名单或版本号机制）。

---

## 2. 数据安全

- 密码使用 bcrypt（cost ≥ 10）或 argon2id 哈希存储，禁止明文存储。
- 所有业务数据通过 workspace_id 进行隔离，查询时自动附加过滤条件。
- 敏感操作（重置密码、解散空间、删除角色）记录详细操作日志。
- 全链路使用 HTTPS / WSS。

---

## 3. 防攻击

- 登录接口设置频率限制（同一 IP 每分钟 5 次）。
- 邀请加入接口设置 CSRF 防护。
- 前端对用户输入进行 XSS 转义处理。
- API 参数校验在服务端完整执行，前端校验仅作为辅助。

---

## 4. 日志审计

- 写操作（创建、更新、删除）记录操作日志，包含：操作人、时间、IP、操作类型、变更内容。
- 日志不记录密码、Token 等敏感信息。
- 后端使用 SLF4J + Logback，按天滚动，保留 30 天。
